# Plan: Combat System — Firing, Death & Enemy AI

Adds a working combat loop: soldiers fire bullets (player via right-click,
enemies via AI), bullets hit soldiers using asymmetric hitboxes, and killed
soldiers play a death animation before removal. Builds on the existing `Soldier`
hierarchy, the skeletal `bullet.dart`, and the spec in `docs/ENEMY_AI_SPEC.md`.

All animation groups (standing-with-gun, death variants, throw) are loaded from
the `junarmy.json` atlas using **semantic sprite names** (e.g.
`player_walk_s`, `enemy_firing_ne`, `bullet`) generated by `fodder_tools`.
The `SoldierState` enum is expanded, `SoldierAnimations` gains new loaders, and
`EnemySoldier` gets a real AI state machine. Each step is an incremental,
testable commit.

## Deferred (out of scope)

- Grenade throwing (1/32 chance, arcing projectile, range < 130 px)
- Rocket soldiers (type 36, rockets only, separate animation groups)
- Dynamic spawning from buildings (`BuildingDoor` component)
- Win conditions / phase objectives
- Squad formation (multiple player soldiers)
- Sound effects
- Camera panning

## Decisions

- **No health system**: one-hit kills per spec; `isAlive` bool suffices
- **Right-click = fire, left-click/tap = move**
- **Speed model**: spec ticks × 5.0 = px/s (e.g. aggression 6 → 18 × 5 = 90)
- **Death anim simplified**: play death sprite + fade; skip full body-arc physics
  initially
- **Grenades + rockets deferred**: focus on gun combat first
- **Standing-with-gun as firing pose**: original has no separate firing animation
- **Semantic sprite names**: atlas frames use `player_walk_s`, `bullet`, etc.
  instead of hex group IDs — generated by `fodder_tools/sprite_names.dart`
- **PLAN.md location**: root (replaces completed movement plan)
